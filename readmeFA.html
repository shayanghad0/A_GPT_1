1) معرفی پروژه

این پروژه یک اپلیکیشن تحت وب/پایتونی است که به شما اجازه می‌دهد به‌صورت ساده و تعاملی سؤال بپرسید و پاسخ دریافت کنید. رابط کاربری پروژه با HTML، CSS و JavaScript طراحی شده و بخش سرور با Python مدیریت می‌شود.

هدف این فایل: کمک به کاربر برای نصب سریع، اجرای بدون دردسر و استفاده از برنامه به زبان فارسی.

2) پیش‌نیازها

  • نصب بودن Python 3.10+
  • دسترسی به pip
  • ترجیحاً استفاده از virtual environment
  • مرورگر مدرن مثل Chrome، Edge یا Firefox
اگر در سیستم شما چند نسخه Python نصب است، بهتر است از python3 یا py استفاده کنید.

3) نصب pip و وابستگی‌ها

3-1) بررسی نصب بودن Python

python --version
pip --version

3-2) اگر pip نصب نبود

در بیشتر نسخه‌های جدید Python، pip بهcopy-btn" onclick="copyCode(this)">کپیpython -m venv-btn" onclick="copyCode(this)">کپیpython -m ensurepip --upgrade

3-3) ساخت محیط مجازی پیشنهادی

python -m venv venv

3-4) فعال‌سازی محیط مجازی

ویندوز

venv\Scripts\activate

لینوکس / مک

source venv/bin/activate

3-5) نصب کتابخانه‌ها

اگر فایل requirements.txt دارید: 

pip install -r requirements.txt

اگر فایل requirements ندارید، کتابخانه‌های موردنیاز را بر اساس کد پروژه نصب کنید. برای پروژه‌های وب پایتونی معمولاً مواردی مثل Flask یا FastAPI رایج هستند.

پیشنهاد می‌شود همیشه نصب وابستگی‌ها را داخل محیط مجازی انجام دهید تا تداخل با پروژه‌های دیگر ایجاد نشود.

4) اجرای پروژه

  1. وارد پوشه پروژه شوید.
  2. محیط مجازی را فعال کنید.
  3. وابستگی‌ها را نصب کنید.
  4. فایل اصلی برنامه را اجرا کنید.

نمونه اجرای معمول

python app.py

بعد از اجرا، معمولاً آدرس برنامه در ترمینال نمایش داده می‌شود، مانند:

http://127.0.0.1:5000
اگر برنامه روی پورت دیگری اجرا شد، همان آدرس نمایش‌داده‌شده در ترمینال را در مرورگر باز کنید.

5) نحوه پرسیدن سؤال از برنامه

پس از باز شدن صفحه اصلی، معمولاً یک کادر متن برای وارد کردن سؤال وجود دارد. کافی است:

  1. سؤال خود را داخل کادر بنویسید.
  2. روی دکمه ارسال یا پرسش کلیک کنید.
  3. منتظر بمانید تا پاسخ نمایش داده شود.

نمونه سؤال‌های مناسب

نمونه 1:

«سلام، لطفاً نحوه نصب پروژه را توضیح بده.»

نمونه 2:

«چطور برنامه را روی سیستم خودم اجرا کنم؟»

نمونه 3:

«چرا پروژه اجرا نمی‌شود و خطا می‌دهد؟»

نمونه 4:

«نحوه استفاده از رابط کاربری را توضیح بده.»

اگر پاسخ‌ها به مدل/سرور خارجی وابسته باشند، اتصال اینترنت یا تنظیمات کلید API نیز ممکن است لازم باشد.

6) ساختار پروژه

ساختار رایج این پروژه به شکل زیر است:

agpt_project/
├── app.py
├── templates/
│   └── index.html
└── static/
    ├── style.css
    └── script.js
  • app.py : فایل اصلی بک‌اند و منطق برنامه
  • templates/index.html : قالب اصلی صفحه
  • static/style.css : استایل‌ها و طراحی ظاهری
  • static/script.js : رفتارهای تعاملی سمت مرورگر

7) عیب‌یابی

مشکل: pip شناخته نمی‌شود

راه‌حل:

python -m pip install --upgrade pip

مشکل: ماژول پیدا نشد

مثال:

ModuleNotFoundError: No module named 'flask'

راه‌حل:

pip install flask

مشکل: صفحه باز می‌شود ولی پاسخی نمی‌دهد

  • کنسول مرورگر را بررسی کنید.
  • ترمینال/لاگ سرور را نگاه کنید.
  • مطمئن شوید سرور backend در حال اجراست.
  • اگر API استفاده می‌شود، کلید و تنظیمات را بررسی کنید.

8) سوالات متداول

آیا باید از محیط مجازی استفاده کنم؟

بله، بسیار توصیه می‌شود چون وابستگی‌ها را جدا نگه می‌دارد و از تداخل جلوگیری می‌کند.

اگر برنامه اجرا نشد چه کنم؟

ابتدا نسخه Python، نصب pip، نصب وابستگی‌ها و سپس لاگ خطا را بررسی کنید.

آیا می‌توانم پروژه را روی سیستم دیگری هم اجرا کنم؟

بله، کافی است Python و وابستگی‌های موردنیاز را نصب کنید و مراحل اجرا را تکرار کنید.

چطور سؤال به فارسی بپرسم؟

فقط متن سؤال خود را فارسی در کادر ورودی تایپ کنید و دکمه ارسال را بزنید.
